# i18n / Language Support Design

## Overview

Add internationalization (i18n) to vvrite: UI language selection, ASR recognition language selection, and multilingual README documentation.

## Supported Languages (14)

All languages are a subset of Qwen3-ASR's `support_languages`. The same list is used for both UI language and ASR language selection.

| Code | Native Name | English Name | ASR `language` param |
|------|-------------|--------------|----------------------|
| `en` | English | English | `"English"` |
| `ko` | 한국어 | Korean | `"Korean"` |
| `ja` | 日本語 | Japanese | `"Japanese"` |
| `zh-Hans` | 简体中文 | Chinese (Simplified) | `"Chinese"` |
| `zh-Hant` | 繁體中文 | Chinese (Traditional) | `"Chinese"` |
| `es` | Español | Spanish | `"Spanish"` |
| `fr` | Français | French | `"French"` |
| `de` | Deutsch | German | `"German"` |
| `pt` | Português | Portuguese | `"Portuguese"` |
| `ru` | Русский | Russian | `"Russian"` |
| `ar` | العربية | Arabic | `"Arabic"` |
| `hi` | हिन्दी | Hindi | `"Hindi"` |

Note: `zh-Hans` and `zh-Hant` both map to `"Chinese"` for ASR. The distinction is UI-only.

### `t()` function

```python
def t(key: str, **kwargs) -> str:
    """
    Translate a dot-separated key path.
    Looks up in current locale dict, falls back to English.
    Supports format parameters: t("menu.hotkey", hotkey="⌥Space")
    """
```

- Fallback chain: selected locale -> English
- Missing key returns the key itself (defensive)
- Format parameters via `str.format(**kwargs)`

### Locale code to module mapping

Locale codes with hyphens (e.g. `zh-Hans`) map to Python modules with underscores (e.g. `zh_hans.py`). The `__init__.py` handles this: `code.replace("-", "_")` to derive the module name.

### Locale resolution on startup

1. Check `NSUserDefaults` for `ui_language`
2. If `None` (System Default): read `NSLocale.preferredLanguages()[0]`, match against `SUPPORTED_LANGUAGES`
3. If no match: fall back to `"en"`

### Initialization order

`set_locale()` must be called as the **very first action** in `applicationDidFinishLaunching_`, before constructing `StatusBarController`, `OverlayController`, or any other UI objects. This ensures all `t()` calls during UI construction return correctly localized strings.

### Thread safety

`_current_locale` is only mutated on the main thread (via `set_locale()`). `t()` is safe to call from any thread because Python dict reads are GIL-protected. No locking is needed.

## Preferences

Two new keys in `NSUserDefaults` (via `preferences.py`):

| Key | Default | Display | Description |
|-----|---------|---------|-------------|
| `ui_language` | `None` | "System Default" | UI display language |
| `asr_language` | `"auto"` | "Automatic" | ASR recognition language |

Both stored in `NSUserDefaults` (`~/Library/Preferences/`), so they persist across DMG-based app updates.

Both keys must be added to `_PREFERENCE_KEYS` in `preferences.py` (alongside `mic_device`) to ensure legacy preference migration works correctly.

### ASR language integration

In `transcriber.py`'s `transcribe()`:

```python
asr_lang = prefs.asr_language  # e.g. "ko" or "auto"
if asr_lang == "auto":
    language_param = None  # model auto-detects
else:
    language_param = ASR_LANGUAGE_MAP.get(asr_lang)
    if language_param is None:
        # Unknown code (corrupted pref, etc.) — log warning, fall back to auto-detect
        print(f"Unknown ASR language code: {asr_lang}, falling back to auto-detect")

result = _model.generate(normalized_path, language=language_param, **kwargs)
```

## UI Changes

### Onboarding — Step 1 (Welcome)

Add a language selection dropdown below the subtitle:

- Items: "System Default", "English", "한국어", "日本語", ...
- Always displayed in native names (readable regardless of current language)
- On change: save to preferences, call `set_locale()`, then call `_show_step(self._step)` to rebuild the current step with new strings. The persistent navigation buttons ("Back", "Get Started"/"Next"/"Done") must also be updated since they live outside the content area.

### Settings — New "Language" section

Added near the top of the settings window (after or before Shortcut):

| Label | Control |
|-------|---------|
| UI Language | Dropdown: System Default, English, 한국어, 日本語, ... |
| Recognition Language | Dropdown: Automatic, English, 한국어, 日本語, ... |

- **UI Language change**: show restart-required dialog ("Restart vvrite to apply the new language." with "Restart Now" / "Later" buttons). Also invalidate the cached settings window (`_settings_wc = None` in AppDelegate) so the next open rebuilds with new strings.
- **Recognition Language change**: takes effect on next recording, no restart needed

### Status Bar Menu

All menu items translated via `t()`:
- `t("menu.settings")` → "Settings..." / "설정..."
- `t("menu.check_updates")` → "Check for Updates..." / "업데이트 확인..."
- `t("menu.quit")` → "Quit vvrite"
- `t("status.ready")` → "Ready" / "준비"
- etc.

**Important:** `setStatus_()` currently compares raw strings for icon state (`_update_icon` checks against `"Ready"`, `"Recording..."`, etc.). This must be refactored: `setStatus_` should accept a status key (e.g. `"ready"`, `"recording"`) and translate internally, using the key for icon logic instead of the translated string.

### Overlay

- `t("overlay.transcribing")` → "Transcribing..." / "받아쓰는 중..."

### Alerts

All alert titles, messages, and button labels translated via `t()`.

## README Multilingual Support

- `README.md` remains in English (primary)
- Language switcher links at top: `🌐 [한국어](README.ko.md) | [日本語](README.ja.md) | ...`
- Separate files for major languages only (7 of 14): `README.ko.md`, `README.ja.md`, `README.zh-Hans.md`, `README.zh-Hant.md`, `README.es.md`, `README.fr.md`, `README.de.md`. Remaining languages (`pt`, `ru`, `ar`, `hi`, `tr`, `it`) may be added later as needed.
- Same structure as English README, LLM-generated initial translations
- Each translated README also includes reciprocal language switcher links to all other languages and back to English

## Translation Authorship

All 14 languages will have LLM-generated initial translations, to be refined over time.
